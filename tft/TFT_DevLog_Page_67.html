12:09:40RESEARCH
I've been running a test with zipstream in order to see if I can make JCR6 zip compatible. The problem is that zip files are read fine, but the filelists cannot be retrieved, and JCR6 relies pretty heavily on that. In fact it's one of its core features in order to make patching and stuff possible, which is of course something the creators of zipstream or of the zip file format in general never took into account, this is where JCR6 can get its sophistication from. However never fear, there are ways in making this still possible, however it will require the zip files to contain a file called "JCR6/Index". It will make zip support way more fragile, however it will make zip also able to take advantage of features JCR6 provides. A tool allowing me (and you) to do this quickly can be in the making ;)
