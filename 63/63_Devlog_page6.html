13:04:50STATUSWith that the moment has come to focus on the savegame routine first.
I'm not 100% sure what is the best way to do this. I want to prevent savegames to have multiple files. In Star Story, Dyrt and TFT REVAMPED the savegames are all multiple files, but as the savegame routine packed them all automatically in one big JCR file, no harm done, as in the end we still had one file. Now this game already uses a dependency in order to read JCR files, but Ryanna has been set up to avoid all that if really needed. That was for the game data an easy thing to do. Savegames are a bit harder in this perspective. Getting the dependency to collect all the data is a problem point for starters, also making the savegame routine able to go for an alternate routine in distributions that do not use this dependency can destabalise stuff. Reading savegames that are pure JCR is no problem. the jcrx dependency picks that up as easily as normal game files, and the Ryanna core scripts already support reading off-project jcr files just fine. JCRX is also scripted to do the same with "real dirs".... Cut shorting loading is not the problem. Saving is....
